# План тестирования (Test Plan) для проекта "TaskManager"

## 1. Введение

Данный документ описывает план тестирования мобильного приложения "TaskManager" для Android.

**Цель:** Проверить соответствие всех функциональных и нефункциональных требований, стабильность и готовность к выпуску.

## 2. Объект тестирования

**Приложение:** "TaskManager" — мобильное приложение для управления задачами с календарем, уведомлениями и Telegram-интеграцией.

**Основные модули:**
- Управление задачами (создание, редактирование, удаление, подзадачи)
- Календарная интеграция (месячный/недельный вид, повторяющиеся задачи)
- Система уведомлений (push-уведомления, напоминания)
- Организация (приоритеты, категории, папки, прогресс)
- Telegram-интеграция
- Автономная работа и синхронизация
- Хранение данных (SQLite, резервные копии, шифрование)

## 3. Риски

- Потеря данных при сбоях
- Проблемы с производительностью (>5000 задач)
- Crash при некорректном вводе
- Ошибки календаря (повторяющиеся задачи, перенос)
- Сбои push-уведомлений
- Проблемы с отображением на разных экранах

## 4. Функциональное тестирование

### Управление задачами
- ✓ Создание, редактирование, удаление задач
- ✓ Подзадачи и чек-листы
- ✓ Отметка выполнения

### Календарь
- ✓ Месячный и недельный вид
- ✓ Drag-and-drop перенос задач
- ✓ Повторяющиеся задачи (ежедневно, еженедельно, ежемесячно)

### Уведомления
- ✓ Push-уведомления на время
- ✓ Настройка напоминаний
- ✓ Уведомления о дедлайнах (24ч, 1ч)

### Организация
- ✓ Приоритеты (высокий, средний, низкий)
- ✓ Категории и цвета
- ✓ Папки и проекты
- ✓ Прогресс-индикаторы

### Telegram
- ✓ Создание/управление задачами через бота
- ✓ Синхронизация с приложением

### Автономная работа
- ✓ Работа без интернета
- ✓ Фоновая синхронизация при появлении соединения
- ✓ Разрешение конфликтов

## 5. Нефункциональное тестирование

### UI/Usability
- Интуитивность интерфейса
- Адаптивность (4"-7" экраны, портрет/ландшафт)
- Поддержка светлой/темной темы
- Основные функции за 3 клика

### Производительность
- Запуск приложения ≤ 3 сек
- Время отклика ≤ 1 сек
- Стабильная работа с 10,000 задач
- Оптимальное потребление памяти/батареи

### Надежность
- Отсутствие crash при нормальной работе
- Корректная работа при повороте экрана
- Обработка ошибок без потери данных

### Безопасность
- Шифрование SQLite базы
- Автоматическое резервное копирование
- Целостность данных при некорректном завершении

## 6. Подходы и инструменты

**Виды тестирования:**
- Функциональное (позитивное + негативное)
- Usability
- Performance (время запуска, отклик, нагрузка)
- Compatibility (разные версии Android, экраны)
- Security
- Regression
- Integration (Telegram API, Firebase)

**Инструменты:**
- Android Studio (эмуляторы: Pixel 5 API 34, Pixel 3a API 24)
- Физические устройства (3+ модели)
- Android Profiler (производительность)
- Logcat (логи/отладка)
- Postman (Telegram API)
- GitHub (документация)

## 7. Критерии Pass/Fail

| Статус | Описание |
|--------|----------|
| **Pass** | Результат совпадает с ожиданием, нет критических ошибок |
| **Fail** | Результат не совпадает, crash, потеря данных |
| **Blocked** | Невозможно выполнить (зависит от другого теста) |
| **Skipped** | Функция еще не реализована |

### Приоритеты дефектов

| Приоритет | Примеры |
|-----------|---------|
| **Critical** | Crash при запуске, потеря всех задач, no задач в календаре |
| **High** | Не работают уведомления, не сохраняются изменения |
| **Medium** | Проблемы UI, задержка синхронизации |
| **Low** | Опечатки, мелкие дизайн-проблемы |

### Критерии готовности к релизу

✓ Критические и High дефекты устранены  
✓ Pass rate ≥ 95%  
✓ Требования производительности выполнены  
✓ Тестирование на всех целевых устройствах  
✓ Отсутствует потеря данных  
✓ Usability рейтинг ≥ 4/5
